RF - requisito funcional
- [ ] O Usuário deve poder criar uma nova transacao
- [ ] O usuário deve poder obter um resumo da sua conta.
- [ ] O usuário deve poder visualizar uma transacao unica.
- [ ] O usuario deve poder listar todas as transacoes que ja ocorreram

RN - regra de negocio

- [ ] A transacao pode ser do tipo crédito que somará ao valor total,
 ou ira subtrari o debito
- [ ] Deve ser possivel identificar o usuario entre as requisições.
- [ ] O usuario so pode visualizer transacao que ele criou

RNF - requisito nao funcional
